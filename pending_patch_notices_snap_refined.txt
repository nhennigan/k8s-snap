- Address vulnerable Go and Python dependencies in the 1.32 snap to reduce exposure to known security issues ([#2483](https://github.com/canonical/k8s-snap/pull/2483)).
- Add a documented revert path for failed upgrades so operators can recover more predictably when an upgrade does not complete cleanly ([#2449](https://github.com/canonical/k8s-snap/pull/2449)).

- Switch Cilium agent and operator references to static image tags for more predictable upgrades and rollbacks ([#2325](https://github.com/canonical/k8s-snap/pull/2325)).
- Add a k8s-dqlite deprecation warning during bootstrap so operators are alerted earlier to the datastore transition ([#2340](https://github.com/canonical/k8s-snap/pull/2340)).
- Add node-join revert handling to improve rollback safety when a join operation fails ([#2337](https://github.com/canonical/k8s-snap/pull/2337)).
- Disable unsupported volume resize for local storage to avoid failed or misleading resize behavior ([#2248](https://github.com/canonical/k8s-snap/pull/2248)).
- Fix invalid `iproute2` JSON handling on arm64 nodes to improve reliability of networking-related operations ([#1821](https://github.com/canonical/k8s-snap/pull/1821)).
- Prevent the feature controller from running on worker nodes to avoid unnecessary reconciliation and instability outside the control plane ([#1938](https://github.com/canonical/k8s-snap/pull/1938)).
- Sanitize feature gates that are not applicable during configuration so unsupported settings do not trigger unnecessary failures ([#2023](https://github.com/canonical/k8s-snap/pull/2023)).
